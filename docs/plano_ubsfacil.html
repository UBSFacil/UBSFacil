📌 Visão Geral

O que é o UBSFacil?

O UBSFacil é uma plataforma web voltada para cidadãos e pacientes que utilizam o sistema público de saúde. O objetivo é facilitar o acesso a informações essenciais — sem precisar ir pessoalmente até uma UBS só para descobrir se tem o medicamento disponível ou se há vaga no atendimento.

  • 📅Consulta de atendimento disponível: visualização de horários e vagas disponíveis em UBSs próximas
  • 🔍Disponibilidade de medicamentos: busca por medicamento e unidade, com estoque atualizado em tempo real
  • 📍UBSs próximas: localização e informações das unidades de saúde com base na localização do usuário
  • 🔔Notificações: alertas quando um medicamento específico estiver disponível ou quando o estoque estiver acabando
💡

Propósito social: O UBSFacil reduz deslocamentos desnecessários, economiza tempo dos pacientes e contribui para um sistema de saúde mais eficiente e acessível.

🔄 Workflow

Fluxo de Trabalho da Equipe

Siga sempre este fluxo para qualquer funcionalidade nova — do planejamento ao deploy. Isso garante qualidade, rastreabilidade e menos dor de cabeça na integração!

  • 1️⃣Planejamento: defina a task no backlog com critérios de aceite claros — o que precisa funcionar para considerar pronto?
  • 2️⃣Desenvolvimento: cada squad trabalha em branches separadas — nunca diretamente na branch main
  • 3️⃣Code Review: abra um Pull Request e peça revisão de pelo menos 1 outro membro da equipe antes do merge
  • 4️⃣Testes: rode os testes unitários e de integração antes de qualquer merge — sem testes quebrando!
  • 5️⃣Staging: faça o deploy em ambiente de homologação e valide o funcionamento com a equipe
  • 6️⃣Produção: deploy controlado com plano de rollback caso algo dê errado
# Fluxo resumido no terminal git checkout -b feature/consulta-medicamentos # ... desenvolve a funcionalidade ... git add . && git commit -m "feat: adiciona rota de consulta de medicamentos" git push origin feature/consulta-medicamentos # abre Pull Request no GitHub → code review → merge!

💡 Dicas

Dicas de Ouro para o Time

  • 🚫Nunca faça push direto na main — sempre crie uma branch e abra um Pull Request, sem exceção!
  • 🔑Jamais suba credenciais no Git — use variáveis de ambiente com .env e adicione ao .gitignore
  • 💬Commits com mensagens claras — use o padrão Conventional Commits: feat:, fix:, docs:, refactor:
  • 📖Documente as rotas da API — o FastAPI gera documentação automática em /docs. Mantenha as descrições atualizadas!
  • 🧪Teste antes de pedir review — rode os testes localmente antes de abrir o Pull Request. Economiza tempo de todos!
  • 📡Comunique bloqueios rápido — se travar em algo por mais de 2h, avise no canal da equipe. Pedir ajuda é sinal de maturidade!
